# Code Readability, Component Architecture, and Business Intent

## Clean Code Practices
- Write highly readable code accompanied by concise, helpful comments—especially documenting the purpose and inputs of individual functions.
- Enforce absolute type consistency across the entire Next.js application using TypeScript.

## Component Architecture
- Prevent monolithic files. Break complex UI views down into modular, reusable sub-components located in separate files within the `components/` directory.

## Business Mindset
- Always approach development through the lens of the final user: a non-tech-savvy store manager trying to efficiently run a small business and organize employee hours. Keep interfaces clean, forgiving, and simple to navigate.
- Ensure all custom styling elements strictly pull from the defined Red Bean brand color palette.
